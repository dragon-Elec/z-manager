### Initialize Libadwaita with AdwApplication (C)

Demonstrates initializing Libadwaita by using AdwApplication instead of GtkApplication. This is the recommended approach as it handles initialization automatically. The example creates a simple window with a label.

```c
#include <adwaita.h>

static void
activate_cb (GtkApplication *app)
{
  GtkWidget *window = gtk_application_window_new (app);
  GtkWidget *label = gtk_label_new ("Hello World");

  gtk_window_set_title (GTK_WINDOW (window), "Hello");
  gtk_window_set_default_size (GTK_WINDOW (window), 200, 200);
  gtk_window_set_child (GTK_WINDOW (window), label);
  gtk_window_present (GTK_WINDOW (window));
}

int
main (int   argc,
      char *argv[])
{
  g_autoptr (AdwApplication) app = NULL;

  app = adw_application_new ("org.example.Hello", G_APPLICATION_DEFAULT_FLAGS);

  g_signal_connect (app, "activate", G_CALLBACK (activate_cb), NULL);

  return g_application_run (G_APPLICATION (app), argc, argv);
}

```

### Apply default window background and foreground colors (.background)

The `.background` style class sets a widget's background and foreground colors to match the default window colors. This is useful for widgets needing an opaque background. It is equivalent to applying specific CSS variables for background and foreground colors.

```css
.background {
  background-color: var(--window-bg-color);
  color: var(--window-fg-color);
```
